# Docs Foundation

## Objective

Define the project's intent before structure or implementation.

## Core Outputs

- vision
- project overview
- scope
- business context
- product map
- domain map

## Working Rules

- Write from business intent to system implications.
- Keep scope explicit.
- Separate current facts from assumptions.
- Mark unresolved questions clearly.
- Stop before detailed system architecture or repo layout work.

## Placeholders To Fill

- target users
- success measures
- scope boundaries
- core domains
- constraints

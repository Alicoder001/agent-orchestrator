# Governance Workflow

## Objective

Define how the project operates, not just what it builds.

## Core Outputs

- repo governance
- architecture decision flow
- ownership model
- definition of ready
- definition of done
- change management
- agent operating model

## Working Rules

- Prefer simple, enforceable rules.
- Define escalation paths early.
- Keep ownership visible.
- Separate policy from day-to-day status updates.
- Keep governance distinct from per-agent logging and release execution details.

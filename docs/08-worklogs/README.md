# Worklogs

This section tracks execution history for humans and agents.

## Model

- Each agent or workstream should maintain its own log file.
- Shared logs should be summary-oriented, not verbose.
- Updates should be append-only where possible.

## Recommended Files

- `status-board.md`
- `agents/agent-01.md`
- `agents/agent-02.md`
- `workstreams/platform.md`
- `workstreams/architecture.md`

## Rules

- Keep entries short.
- Include date, owner, and outcome.
- Record blockers explicitly.

# Market Thesis

## Maqsad

Ushbu hujjat platformaning bozor gipotezasini — target market hajmi, o'sish trend'lari va timing'ni belgilaydi.

Bu hujjat `business-context.md` (business model), `competitive-landscape.md` va `platform-positioning.md` ga asoslanadi.

---

## Core Thesis

> **2025-2028 orasida har bir dasturiy ta'minot jamoasi AI agent'larni "virtual developer" sifatida ishlatadi. Bu agent'larni boshqarish uchun yangi kategoriya platform kerak.**

## Market Sizing (2025-2028)

### TAM (Total Addressable Market)

| Segment | Hajm | Basis |
|---------|------|-------|
| Global software developers | ~30 million | Evans Data, Stack Overflow |
| AI-assisted development market | $15B (2026) → $45B (2028) | Industry estimates |
| Agent orchestration niche | $1-3B (2028) | Our estimate, 5-10% of AI dev market |

### SAM (Serviceable Addressable Market)

| Segment | Hajm |
|---------|------|
| English-speaking dev teams using AI tools | ~5 million developers |
| Teams with 3+ AI agent tools | ~500K teams (2026 estimate) |
| Revenue potential (SAM) | $500M-$1B (2028) |

### SOM (Serviceable Obtainable Market)

| Phase | Target | SOM |
|-------|--------|-----|
| V1-V2 (Year 1) | Early adopters, solo devs | 1000-5000 users |
| V3 (Year 2) | Small teams, startups | 10K-50K users |
| V5+ (Year 3+) | Medium companies | 50K-200K users |

## Market Trends

### 1. AI Coding Adoption Curve

```
2023: ~10% developers using AI (Copilot early)
2024: ~30% developers using AI (Cursor, Copilot growth)
2025: ~50% developers using AI (mainstream)
2026: ~70% developers using AI (standard tool)
2028: ~90% (AI-native = default)
```

### 2. Agent Shift (Tool → Agent)

```
2024: AI = Autocomplete (code completion)
2025: AI = Junior Dev (full task completion)
2026: AI = Team Member (project-level contribution)
2028: AI = Managed Workforce (orchestrated, monitored)
```

### 3. Organizational AI

```
2024: Individual adoption (developer-driven)
2025: Team adoption (tech lead-driven)
2026: Org adoption (CTO-driven, policy, governance)
2028: AI-native organizations (built around AI from day 1)
```

## Timing Argument

| Nima uchun HOZIR? |
|-------------------|
| AI coding agent'lar birinchi marta production-ready (Cursor, Devin) |
| "Attention problem" birinchi marta haqiqiy muammo — ko'p agent = ko'p terminal |
| Multi-agent orchestration platformasi **hali mavjud emas** |
| Early mover advantage — kategoriya yaratish imkoniyati |
| Open-source community momentum — developer trust qurish vaqti |
